Claude Tmux Plugin

Interactive demonstrations of parallel agents, background services, and tmux integration

Version 1.0.0
Demo 1

Detect Tmux Session

Check your current tmux environment and get detailed session information including session name, window index, and pane coordinates.
Use case: Understanding your tmux environment
Command: detect-session.sh
Demo 2

Spawn Visible Pane

Create a split pane for parallel work the user can see. Perfect for running multiple Claude agents or monitoring live output side-by-side.
Use case: Parallel Claude agents
Command: spawn-pane.sh -d h -c "command"
Demo 3

Spawn Background Window

Start a background window for servers, watchers, and long-running processes. Keeps them out of the way while still running.
Use case: Background services and servers
Command: spawn-window.sh -n "server" -c "npm run dev"
Demo 4

Wait for Pattern

Wait for a server to be ready or a process to complete by monitoring for specific output patterns. Essential for orchestrating dependent tasks.
Use case: Task orchestration and waiting
Command: wait-for-text.sh -t "window:0.0" -p "ready" -T 30
Demo 5

Capture Output

Capture and display output from any pane without switching windows. Get JSON-formatted data with metadata for programmatic use.
Use case: Monitoring and status checks
Command: capture-output.sh -t "window:0.0" -l 50 --json
Demo 6

Queue Message for Next Session

Queue reminders and follow-up messages for your next Claude session. Ensures important tasks and notes aren't forgotten.
Use case: Self-continuation and reminders
Command: queue-message.sh "Check test results and deploy"
Demo 7

Complete Workflow

A complete real-world workflow combining all features: spawn server in background, show logs in visible pane, wait for readiness, and queue follow-up.
Use case: Real-world development workflow
Commands: All together (pane + window + wait + message)

Getting Started

Want to try these features yourself?

git clone https://github.com/buremba/claude-tmux ~/.claude/plugins/claude-tmux

Then open Claude Code in tmux and you'll see TMUX_ENVIRONMENT_DETECTED in your context!
